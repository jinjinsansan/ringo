## 2. 期待値分析

### 各紹介レベルでの期待値計算

#### 紹介人数0人時の期待値

```
E = (60% × 0) + (25% × -1) + (10% × -2) + (3% × -9) + (2% × +1)
  = 0 - 0.25 - 0.20 - 0.27 + 0.02
  = -0.70
```

**解釈**：平均的に、ユーザーは1回のりんご抽選で0.70回分、「購入してもらえる権利」を失う。言い換えれば、**購入義務の方が重い**。

#### 紹介人数1人時の期待値

```
E = (50% × 0) + (30% × -1) + (15% × -2) + (4% × -9) + (1% × +1)
  = 0 - 0.30 - 0.30 - 0.36 + 0.01
  = -0.95
```

**解釈**：期待値が悪化している。これは問題。

#### 紹介人数2人時の期待値

```
E = (40% × 0) + (35% × -1) + (18% × -2) + (5% × -9) + (2% × +1)
  = 0 - 0.35 - 0.36 - 0.45 + 0.02
  = -1.14
```

**解釈**：さらに悪化。紹介すればするほど、期待値が悪くなる。これは逆効果。

#### 紹介人数3人以上時の期待値

```
E = (30% × 0) + (35% × -1) + (25% × -2) + (8% × -9) + (2% × +1)
  = 0 - 0.35 - 0.50 - 0.72 + 0.02
  = -1.55
```

**解釈**：最悪。紹介人数が増えるほど、期待値が悪くなる。

## 3. 現在の設計の問題点

### 問題1：紹介インセンティブが逆効果

**現在の設計**：紹介人数が増えると、上位りんご（赤いりんご、ゴールド）の確率が上がる。

**しかし、期待値分析では**：紹介人数が増えるほど、期待値が悪くなる。

**ユーザー心理**：
- 「紹介すれば、上位りんごを引ける確率が上がる」と思う
- しかし、実際には「購入義務が重くなる」
- ユーザーは損をしている

### 問題2：毒りんごの位置付けが不明確

**毒りんご**：1回購入 + 0回購入してもらえる

**期待値**：+1（唯一の正の期待値）

**ユーザー心理**：
- 毒りんごは「ハズレ」だと思う
- しかし、期待値では最も良い結果
- 矛盾している

### 問題3：システムの持続可能性

**長期的には**：
- ユーザーは購入義務を果たす必要がある
- しかし、期待値が負なので、購入権が不足する
- システムが破綻する可能性がある

## 4. 確率論的に破綻しないシステムの設計

### 原則1：全体の期待値は0に近い

**理由**：
- 購入義務と購入権のバランスが取れていないと、システムが破綻する
- 全ユーザーの購入義務の合計 ≈ 全ユーザーの購入権の合計

### 原則2：紹介インセンティブは明確

**理由**：
- ユーザーが紹介する動機を作る
- 紹介すればするほど、メリットが増える

### 原則3：毒りんごの位置付けを明確にする

**理由**：
- ユーザーの期待値とゲーム性のバランスを取る


## 15. 重要な認識：ゼロサムゲームではない

### 現在の理解の誤り

**従来の考え**：「購入義務と購入権のバランスが取れていないと、システムが破綻する」

**実際**：
- 新規ユーザーが参加するたびに、購入権が増える
- 既存ユーザーが購入義務を果たすことで、新規ユーザーが購入権を得る
- **ゼロサムゲームではなく、ポジティブサムゲーム**

### 正しい理解

**システムの持続可能性の条件**：
- 新規ユーザーの参加速度 ≥ 既存ユーザーの購入義務の完了速度

**つまり**：
- ユーザーが増え続ければ、システムは持続する
- ユーザーが増えなくなると、購入権が余ってシステムが停滞する
